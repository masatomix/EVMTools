日替わり処理をJenkinsにやってもらう場合は、チェックを入れてください。 ヒトがJenkinsに対して「日替わりして！」というリクエストを送信することで、Jenkinsが日替わり処理を実施してくれます。

チェックを入れない場合は、先日分ファイル(base_schedule.xls)などはヒトが手動で作成してください。 手動で行うのが煩雑ですが、その代わりに日替わりを緩く自由に管理できます。

日替わり処理をJenkinsにやってもらう場合は「日替わりをしないで次の基準日のスケジュールがコミットされたりするとエラーになる」など、 ある程度のチェック処理が行われます。そのチェック処理が逆にうっとうしいケースなどは、ヒトが直接管理するのがよいと思います。

「日替わりして！」というリクエストですが、Jenkinsのコマンド実行の仕組みを利用します。 Jenkinsにアクセスして
Jenkinsの管理 >> Jenkins CLI を開くとヘルプが参照できますが、具体的には

  • java -jar jenkins-cli.jar -s http://localhost:8080/jenkins higawari EVMTools project_management_tools.xls
などとすることで日替わり処理が実行されます。

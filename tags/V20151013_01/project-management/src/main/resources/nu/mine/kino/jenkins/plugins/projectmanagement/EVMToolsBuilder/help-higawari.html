日替わり処理をJenkinsにやってもらう場合は、チェックを入れてください。 ヒトがJenkinsに対して「日替わりして！」というリクエストを送信することで、Jenkinsが日替わり処理を実施します。

チェックを入れない場合は、先日分ファイル(base_schedule.xls)などはヒトが手動で作成してください。 手動で行うのが煩雑ですが、その代わりに日替わりをユルく、自由に管理できます。

日替わり処理をJenkinsにやってもらう場合は、ある程度のチェック処理が行われます。たとえば「日替わり処理をしないで、次の基準日のスケジュールがコミットされると集計がエラーになる」などです。 そのチェック処理が逆にうっとうしい場合などは、ヒトが手動で管理するのがよいと思います。ちなみにこの場合は日替わり処理を実施するか、元の基準日のファイルを再コミットすればOKです。

「日替わりして！」というリクエストですが、Jenkinsのコマンド実行の仕組みを利用します。 Jenkinsにアクセスして
Jenkinsの管理 >> Jenkins CLI を開くとコマンドの一覧などのヘルプが参照できますが「higawari」コマンドが該当のコマンドです。具体的には

  • java -jar jenkins-cli.jar -s http://localhost:8080/jenkins higawari EVMTools(ジョブ名) project_management_tools.xls(ファイル名)
などとすることで、Jenkinsが日替わり処理を実施します。
